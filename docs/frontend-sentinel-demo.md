# Z-Clear 前端合规检索集成演示文档 (Sentinel Agent Integration)

本文档说明了如何在前端直观地演示 Z-Clear 系统的“实时政策哨兵”（Sentinel Agent）功能，尤其是其核心的 RAG 防幻觉特性。

## 一、新增功能说明

在原有发票提取功能（`POST /extract`）的基础上，本次前端迭代无缝接入了合规政策检索（`POST /sentinel/retrieve`），新增以下两大模块：

1. **自动合规检索展示（自动化主流程集成）**：
   在单据提取完成并解析出 HS 编码和品名后，系统将自动发起合规政策检索。在提取结果下方，新增了「合规政策匹配结果」面板，动态展示合规上下文与检索到的 Top 3 政策原文。期间，状态徽章会动态流转（EXTRACTING -> RETRIEVING -> AUDITING）。
2. **手动合规检索面板（防幻觉专项演示）**：
   在页面下方新增了独立的「合规政策手动检索」区域。用户可以手动输入任意品名、HS 编码和国家信息，直观测试系统的响应。

## 二、演示流程建议

### 场景一：标准合规自动化流程（主流程演示）
1. 打开 Z-Clear 系统页面，语言选择中文（或英文/马来语，系统将自动动态适配）。
2. 在“发票文本”框中输入真实的单据信息（例如包含钢管、HS 编码 `7304` 的发票）。
3. 点击“提取信息”按钮。
4. **观察点 1**：页面上方显示加载中，随后提取结果面板平滑展开，显示品名和 HS 编码。此时状态徽章会从蓝色的“提取中”变为紫色的“检索中...”动态呼吸状态。
5. **观察点 2**：短暂等待后，“合规政策匹配结果”面板自动弹出。展示结构化的“合规上下文”，下方列出匹配到的海关政策。此时右上角的状态徽章会变绿或变蓝为“审计中”/“已完成”。

### 场景二：防幻觉测试（亮点演示）
1. 滚动到页面下方的“合规政策手动检索”区域。
2. 在“商品名称”中输入任意虚假商品（如：`外星飞船引擎`）。
3. 在“HS编码”中输入一个系统中不存在的编码（如：`9999.99.99`）。
4. 点击“检索合规政策”按钮。
5. **观察点**：检索完成后，“合规上下文”区域将明确且如实地显示：`未检索到匹配的海关合规政策或RCEP规则。`。这证明了大模型严格受限于 RAG 检索结果，不会进行任何海关政策的自由伪造（防幻觉）。

## 三、接口对接情况说明
- **提取接口复用**：`POST /extract` 的调用逻辑 100% 复用，入参与出参未作任何修改。
- **检索接口对接**：新增了 `triggerComplianceRetrieval()` 和 `manualRetrieve()` 异步函数，内部完全复用了原生的 `fetch` 封装逻辑。
- **状态管理**：无缝集成了后端下发的 `AUDITING`、`ERROR` 等状态，前端 UI 根据状态字段自动映射对应的背景色与多语言提示。
- **多语言**：前端所有的硬编码字符串（包括新增面板的标题、提示、错误日志）已完全纳入 `translations` 字典，确保 100% 支持中/英/马来语的一键无刷新切换。
